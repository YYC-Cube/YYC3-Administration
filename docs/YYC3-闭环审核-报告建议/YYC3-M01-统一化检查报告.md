## 5. 统一化建议

### 5.1 设计语言统一建议

#### 颜色系统

1. **统一颜色映射来源**：删除NeonCard.tsx中的内联颜色映射，使用`getThemeNavColor`函数
2. **主题化UI组件**：为Button、Card、Input等shadcn/ui组件添加主题适配逻辑
3. **扩展use-theme-tokens**：为LiquidGlass主题添加完整的tokens定义
4. **移除硬编码颜色**：DashboardPage等页面使用`useThemeColors`获取颜色

#### 排版系统

1. **统一字体大小**：严格按照规范使用text-xs(12px)、text-sm(14px)等
2. **统一标题层级**：页面标题使用h1，区块标题使用h2，卡片标题使用h3/h4

#### 组件样式

1. **统一卡片样式**：Card组件添加主题切换支持
2. **统一输入框样式**：确保Input和Textarea的focus状态一致

### 5.2 代码规范统一建议

#### 注释风格

1. **强制JSDoc注释**：所有导出的函数、组件、类型必须使用JSDoc注释
2. **统一文件头部注释**：所有文件添加标准的文件头部注释
3. **统一注释语言**：优先使用中文注释

#### 命名规范

1. **统一文件命名**：UI组件使用kebab-case，业务组件使用PascalCase
2. **统一变量命名**：全部使用camelCase，常量使用UPPER_SNAKE_CASE

#### 导入导出

1. **统一导入顺序**：外部依赖 → 内部依赖 → 样式文件
2. **统一导出方式**：UI组件使用命名导出，业务组件根据情况选择
3. **统一React导入**：使用`import * as React from 'react'`

### 5.3 文档风格统一建议

1. **统一文档格式**：所有文档使用相同的标题层级和目录结构
2. **创建注释模板**：定义组件、函数、类的标准注释格式
3. **创建API文档**：使用Swagger或类似工具生成统一的API文档

### 5.4 交互体验统一建议

1. **统一加载状态**：创建全局Loading组件
2. **统一错误提示**：使用Toast组件作为全局错误提示
3. **统一弹窗行为**：所有弹窗使用Dialog组件或其变体
4. **添加快捷键帮助面板**：创建展示所有快捷键的帮助弹窗
