#### 3.2.1 react-refresh/only-export-components (27 个)

**影响文件**: 主要在 `src/app/components/ui/` 和部分业务组件

**问题**: 一个文件同时导出组件和常量/函数，导致 React Fast Refresh 降级为全量刷新。

**建议修复**:

```tsx
// ❌ 当前: 同一文件导出组件和常量
export const buttonVariants = cva(...)
export const Button = React.forwardRef(...)

// ✅ 改进: 分离到不同文件
// button-variants.ts
export const buttonVariants = cva(...)
// button.tsx
import { buttonVariants } from './button-variants'
export const Button = React.forwardRef(...)
```

> **注意**: 此警告对 shadcn/ui 组件库文件属于设计权衡，仅为开发体验优化，不影响生产构建。建议优先修复业务组件中的此类警告。

### 3.5 死代码检查 — ⚠️ 需确认

**执行命令**: `npx unimported`
**结果**: 报告 113 个文件"未被引用"

**分析**: 此结果大概率是误报，原因如下：

1. unimported 无法识别 Vite 的 `@/*` 路径别名
2. 动态导入 (`import()`) 的模块无法被静态分析
3. 入口文件 `main.tsx` 通过路由间接引用组件

**验证方法**:

```bash
# 方法1: 使用 Vite 构建分析
pnpm build 2>&1 | grep -E "error|warning"

# 方法2: 使用 webpack-bundle-analyzer（已安装）
# 分析打包结果确认是否有未引用模块
```

**建议**: 暂不处理，在 CI/CD 中集成构建产物分析
