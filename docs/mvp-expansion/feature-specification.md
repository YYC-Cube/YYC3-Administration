## F-01: 后端 API 服务层

### 功能描述与目标

构建基于 NestJS 的企业级后端服务层，为所有前端功能提供统一的 RESTful + GraphQL API 接口，支持 WebSocket 实时通信、数据持久化、文件存储和安全认证。

**目标**: 
- 提供高可用 (>99.9%) API 服务
- API 响应时间 P95 < 200ms
- 支持水平扩展
- 统一错误处理和数据校验

### 用户场景与用例

| 场景 | 用户角色 | 用例 |
|------|---------|------|
| 数据请求 | 所有用户 | 登录后通过 API 获取业务数据 |
| 文件上传 | 所有用户 | 上传图片/文档到素材库 |
| 实时通知 | 所有用户 | WebSocket 接收推送通知 |
| 数据导出 | 运营人员 | 通过 API 导出 CSV/JSON 数据 |

### 功能流程图

```
请求流程:
┌─────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 客户端   │───▶│ API 网关  │───▶│ 认证中间件 │───▶│ 控制器   │
└─────────┘    └──────────┘    └──────────┘    └────┬─────┘
                                                     │
                                            ┌────────┴────────┐
                                            │   服务层 Service  │
                                            └────────┬────────┘
                                                     │
                                            ┌────────┴────────┐
                                            │  数据访问层 DAO  │
                                            └────────┬────────┘
                                                     │
                                            ┌────────┴────────┐
                                            │   PostgreSQL     │
                                            └─────────────────┘

认证流程:
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 登录表单  │───▶│ Auth API │───▶│ 验证凭证  │───▶│ JWT 签发 │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
                                                     │
                                              ┌──────┴──────┐
                                              │ 返回 Token   │
                                              └─────────────┘
```

### 技术要求和限制

- **框架**: NestJS 11.x (TypeScript)
- **数据库**: PostgreSQL 16 + Redis 7
- **ORM**: Prisma 6.x
- **API 风格**: RESTful + GraphQL (Apollo)
- **认证**: JWT (access + refresh token)
- **实时通信**: Socket.IO (WebSocket)
- **文件存储**: S3 兼容 (MinIO)
- **任务队列**: Bull (Redis)
- **容器化**: Docker + Docker Compose
- **API 文档**: Swagger/OpenAPI 3.0
- **输入校验**: class-validator + class-transformer
- **错误处理**: 统一异常过滤器

### 接口规范

```
# 通用响应格式
{
  "code": 200,
  "message": "success",
  "data": {},
  "meta": {
    "page": 1,
    "pageSize": 20,
    "total": 100
  }
}

# 错误响应格式
{
  "code": 40001,
  "message": "Validation failed",
  "errors": [
    { "field": "email", "message": "Invalid email format" }
  ]
}
```

## F-02: 用户认证与权限系统

### 功能描述与目标

完整的用户身份认证、授权与权限管理系统，支持多角色、资源级权限控制和操作审计。

**目标**:
- 支持邮箱/手机号 + OAuth2 社交登录
- RBAC + 资源级 ACL 权限模型
- 完整的审计日志
- 密码安全策略 (bcrypt/argon2)

### 用户场景与用例

| UC-ID | 场景 | 用户角色 | 步骤 |
|-------|------|---------|------|
| UC-01 | 用户注册 | 新用户 | 填写邮箱/手机 → 验证码 → 设置密码 → 注册成功 |
| UC-02 | 用户登录 | 注册用户 | 输入凭证 → 验证 → 返回 JWT → 跳转到首页 |
| UC-03 | OAuth2 登录 | 新/老用户 | 选择微信登录 → 授权 → 回调 → 绑定/登录 |
| UC-04 | 角色管理 | 超级管理员 | 创建角色 → 分配权限 → 关联用户 |
| UC-05 | 权限校验 | 系统 | 请求 → 鉴权中间件 → 权限判定 → 放行/拒绝 |
| UC-06 | 查看审计日志 | 安全管理员 | 进入审计页面 → 筛选条件 → 查看操作记录 |

### 权限模型设计

```
RBAC + ACL 混合模型:

用户 ──── 角色 ──── 权限 ──── 资源
 │         │          │         │

 │    │ 管理员   │     │         │
 │    │ 运营     │     │         │

                      │         │
           ┌──────────┴──┐      │
           │ customer:read │     │
           │ customer:write│    │
           │ call:read    │     │
           │ call:execute │     │
           │ finance:read │     │
           │ admin:all    │     │
           └─────────────┘      │
                                │
                    ┌───────────┴───┐
                    │ Customer 实体  │
                    │ Campaign 实体  │
                    │ Finance 实体   │
                    │ Settings 实体  │
                    └───────────────┘

### 界面设计规范

**登录页面**:
- 赛博朋克风格登录背景动画
- 邮箱/手机号输入 + 密码输入
- "忘记密码"链接
- OAuth2 社交登录按钮 (微信/GitHub)
- 表单验证即时反馈

**用户管理页面**:
- 用户列表 (表格 + 搜索 + 筛选)
- 用户详情侧面板
- 角色分配下拉选择
- 状态开关 (启用/禁用)

### 技术要求和限制

- 密码加密: bcrypt (cost=12)
- Token: JWT RS256 (access 15min + refresh 7d)
- 验证码: TOTP 基于时间的一次性密码
- 会话存储: Redis
- 安全防护: Rate limiting, CSRF Token, XSS 过滤
- OAuth2: 支持微信开放平台 + GitHub OAuth App

## F-03: AI 智能体编排引擎

### 功能描述与目标

从单轮对话升级为多智能体协作编排系统，支持智能体的定义、注册、编排、记忆管理和工具调用。

**目标**:
- 支持 5+ 智能体并行协作
- 任务分解准确率 > 85%
- 支持自定义工具/Function Calling
- 上下文窗口管理 (滑动窗口 + 摘要)

### 用户场景与用例

| UC-ID | 场景 | 用户角色 | 步骤 |
|-------|------|---------|------|
| UC-01 | 创建智能体 | 管理员 | 定义名称/描述/模型/指令 → 配置工具 → 保存 |
| UC-02 | 对话路由 | 系统 | 用户输入 → 意图识别 → 路由到对应智能体 |
| UC-03 | 任务分解 | 系统 | 复杂任务 → AI 分解为子任务 → 分配给不同智能体 |
| UC-04 | 工具调用 | 智能体 | 智能体识别需要工具 → 调用 Function → 返回结果 |
| UC-05 | 记忆管理 | 系统 | 对话 → 提取关键信息 → 存储 → 检索 |

### 智能体架构

```
                     ┌──────────────────────┐
                     │    Orchestrator       │
                     │    (编排器)            │
                     └──────┬───────────────┘
                            │
          ┌─────────────────┼──────────────────┐
          │                 │                   │
   ┌──────┴──────┐  ┌──────┴──────┐   ┌──────┴──────┐
   │ Agent A     │  │ Agent B     │   │ Agent C     │
   │ 客服智能体   │  │ 销售智能体   │   │ 数据分析    │
   └──────┬──────┘  └──────┬──────┘   └──────┬──────┘
          │                 │                   │
   ┌──────┴──────┐  ┌──────┴──────┐   ┌──────┴──────┐
   │ Tools:      │  │ Tools:      │   │ Tools:      │
   │ 查询客户     │  │ 生成报价     │   │ SQL查询     │
   │ 创建工单     │  │ 发送邮件     │   │ 图表生成    │
   │ 知识库搜索   │  │ CRM更新     │   │ 报告导出    │
   └─────────────┘  └─────────────┘   └─────────────┘
```

### 界面设计规范

**智能体管理页面**:
- 左侧: 智能体列表 (卡片布局，显示名称/模型/状态)
- 右侧: 智能体编辑面板
  - 基本信息 (名称/描述/头像)
  - 模型选择 (下拉)
  - 系统指令 (富文本编辑器)
  - 工具配置 (开关列表)
  - 知识库关联
  - 测试对话 (嵌入式聊天)

**对话编排界面**:
- 可视化编排图 (React Flow)
- 节点类型: 输入/智能体/工具/条件/输出
- 连线拖拽
- 实时执行追踪

### 技术要求和限制

- AI SDK: Vercel AI SDK + LangChain
- 流式响应: Server-Sent Events (SSE)
- 工具定义: OpenAI Function Calling 格式
- 记忆存储: PostgreSQL (pgvector) + Redis
- 上下文管理: 滑动窗口 (最近 20 轮) + 自动摘要
- 超时控制: 单智能体 30s，编排任务 5min
- 重试策略: 失败自动重试 2 次

## F-04: 自动化工作流引擎 2.0

### 功能描述与目标

从现有可视化工作流升级为可编程+可视化混合引擎，支持条件分支、并行执行、AI 自动生成和执行监控。

**目标**:
- 支持 10+ 节点类型
- AI 工作流生成准确率 > 80%
- 支持定时/事件触发
- 执行日志完整可回溯

### 用户场景与用例

| UC-ID | 场景 | 用户角色 | 步骤 |
|-------|------|---------|------|
| UC-01 | 创建工作流 | 运营 | 拖拽节点 → 配置参数 → 连接节点 → 保存/发布 |
| UC-02 | AI 生成工作流 | 运营 | 描述需求 → AI 自动生成 → 确认/修改 → 发布 |
| UC-03 | 执行工作流 | 系统 | 触发条件满足 → 创建工作流实例 → 按顺序执行 |
| UC-04 | 监控执行 | 运营 | 查看执行列表 → 点击查看详情 → 日志追踪 |

### 节点类型定义

| 节点类型 | 功能 | 配置参数 |
|---------|------|---------|
| **触发器** | 启动工作流 | 手动/定时(cron)/事件(Webhook) |
| **AI 处理** | 调用 AI 模型 | 模型选择/提示词/温度 |
| **条件分支** | 条件判断 | IF/ELSE 条件表达式 |
| **并行** | 并行执行 | 分支数量/合并策略 |
| **API 调用** | HTTP 请求 | URL/方法/Headers/Body |
| **数据转换** | 数据映射 | 输入输出映射 |
| **通知** | 发送通知 | 渠道/模板/接收人 |
| **等待** | 延迟执行 | 等待时长/条件 |
| **子工作流** | 嵌套工作流 | 选择子工作流 |
| **结束** | 结束流程 | 输出/状态回传 |

### 界面设计规范

**工作流设计器**:
- 左侧: 节点工具箱 (分类/搜索/拖拽)
- 中心: 画布区域 (React Flow)
  - 缩放/平移/对齐
  - 迷你地图
  - 节点状态高亮 (运行中/成功/失败)
- 右侧: 节点配置面板 (表单)
- 顶部: 工具栏 (保存/发布/测试/AI生成)

**执行监控页面**:
- 执行列表 (时间/状态/耗时)
- 详情视图 (DAG 执行图 + 日志)
- 重试/终止/回滚操作

### 技术要求和限制

- 可视化库: React Flow 12.x
- 执行引擎: 自研 Workflow Engine (Bull + Redis)
- 表达式引擎: mathjs (安全沙箱)
- 存储: PostgreSQL (工作流定义 + 执行记录)
- 超时: 单步骤 5min，总执行 24h
- 重试: 可配置 (默认 3 次)
- 并发: 每个工作流实例独占执行

## F-11: AI 数据分析引擎

### 功能描述与目标

从现有图表展示升级为 AI 驱动的数据洞察引擎，支持自然语言查询、自动异常检测、趋势预测和智能报告生成。

**目标**:
- NL2SQL 准确率 > 85%
- 异常检测召回率 > 90%
- 支持 5+ 预测算法
- 报告自动生成 < 30s

### 用户场景与用例

| UC-ID | 场景 | 用户角色 | 步骤 |
|-------|------|---------|------|
| UC-01 | 自然语言查询 | 运营 | 输入"上月哪个渠道转化率最高" → NL2SQL → 返回结果 |
| UC-02 | 异常检测 | 系统 | 定时扫描 → 识别异常 → 生成预警 → 推送通知 |
| UC-03 | 趋势预测 | 管理层 | 选择指标 → AI 预测 → 显示置信区间 → 导出 |
| UC-04 | 报告生成 | 运营 | 选择范围 → AI 生成 → 预览/编辑 → 导出 PDF |

### 系统架构

```
用户查询 ──▶ NL2SQL 引擎 ──▶ SQL 生成 ──▶ 数据库 ──▶ 结果
                │                                       │
                │                                       ▼
          意图识别 + 实体提取                      Chart 生成
                │                                       │
                ▼                                       ▼
          查询修正                                    可视化
                │
          反馈学习 ◀──── 用户确认/修改
```

### 界面设计规范

**自然语言查询入口**:
- 查询输入框 (支持自然语言)
- 历史查询记录
- 查询建议 (热门/最近)
- 结果视图自动切换 (表格/图表)

**异常检测视图**:
- 时间轴视图 (标红异常点)
- 异常详情面板 (原因分析/影响范围/建议措施)
- 一键创建工单/告警规则

**报告生成**:
- 报告模板选择
- AI 生成内容预览 (可编辑)
- 章节结构自动组织
- 导出格式: PDF/Word/HTML

### 技术要求和限制

- NL2SQL: LangChain + 自训练模型
- 异常检测: 统计方法 (3-sigma/IQR) + 机器学习 (Isolation Forest)
- 预测算法: ARIMA/Prophet/LSTM (模型选择)
- 时序数据库: TimescaleDB (PostgreSQL 扩展)
- 报告引擎: React-PDF / Puppeteer
- 缓存: 查询结果缓存 5min

## F-12: 智能营销自动化

### 功能描述与目标

AI 驱动的全链路营销自动化系统，支持受众智能分群、多渠道触达编排、A/B 测试和 ROI 实时追踪。

**目标**:
- 受众分群准确率 > 90%
- 支持 6+ 触达渠道
- A/B 测试统计显著
- ROI 追踪实时 < 5s 延迟

### 用户场景与用例

| UC-ID | 场景 | 用户角色 | 步骤 |
|-------|------|---------|------|
| UC-01 | 创建营销活动 | 营销经理 | 选择目标 → 定义受众 → 设计流程 → 设置预算 → 发布 |
| UC-02 | 智能受众分群 | 系统 | 导入客户 → AI 聚类 → 生成分群 → 人工调整 |
| UC-03 | 多渠道触达 | 系统 | 按流程 → 触达各渠道 → 追踪响应 → 优化策略 |
| UC-04 | A/B 测试 | 营销经理 | 定义变量 → 创建版本 → 分配流量 → 分析结果 |

### 营销活动架构

```
                     ┌──────────────────────────┐
                     │    营销活动 Campaign       │
