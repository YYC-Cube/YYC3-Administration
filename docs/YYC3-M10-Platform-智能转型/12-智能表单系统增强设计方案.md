## 12.1 增强目标

基于现有智能表单系统（第三章），新增以下能力维度：

| 维度 | 现有能力 | 增强方向 |
|:-----|:---------|:---------|
| AI 智能化 | 基础字段推荐 | AI 自动生成表单 + 智能校验 + 语义理解 |
| 数据联动 | 单表单数据处理 | 跨表单数据聚合 + 实时数据管道 |
| 业务集成 | 独立表单单元 | 与财务/薪资/CRM 系统深度集成 |
| 审批流 | 基础流程引擎 | 可视化审批流 + 条件分支 + 并行会签 |
| 移动端 | 响应式适配 | 原生级移动体验 + 离线填写 + 拍照上传 |

### 12.2.2 核心接口定义

```typescript
/** 智能表单增强 — 核心类型 */
declare namespace SmartForm {

  // ---- 表单定义 ----
  interface FormDefinition {
    id: string
    name: string
    category: FormCategory
    version: number
    fields: FormField[]
    rules: ValidationRule[]
    workflow?: WorkflowDefinition
    integrations: FormIntegration[]
    aiConfig?: AIFormConfig
    metadata: FormMetadata
  }

  type FormCategory =
    | 'crm'        // 客户管理表单
    | 'finance'    // 财务报销/申请
    | 'hr'         // 人事/薪资
    | 'marketing'  // 营销活动
    | 'operation'  // 运营工单
    | 'custom'     // 自定义

  interface FormField {
    id: string
    type: FieldType
    label: string
    key: string
    required: boolean
    defaultValue?: unknown
    validation?: ValidationRule[]
    visibility?: ConditionExpression
    aiHint?: AIFieldHint
    layout: FieldLayout
  }

  type FieldType =
    | 'text' | 'textarea' | 'number' | 'date' | 'datetime'
    | 'select' | 'multiSelect' | 'cascade' | 'treeSelect'
    | 'fileUpload' | 'imageUpload' | 'signature'
    | 'table' | 'formula' | 'richText' | 'location'
    | 'userPicker' | 'departmentPicker' | 'dataLink'

  interface AIFieldHint {
    autoSuggest: boolean
    ocrEnabled: boolean
    semanticValidation: boolean
    relatedDataSource?: string
  }

  // ---- 审批流 ----
  interface WorkflowDefinition {
    id: string
    nodes: WorkflowNode[]
    edges: WorkflowEdge[]
    variables: WorkflowVariable[]
  }

  interface WorkflowNode {
    id: string
    type: 'start' | 'end' | 'approval' | 'condition' | 'notification' | 'integration'
    name: string
    config: Record<string, unknown>
    assignees?: AssigneeRule[]
    timeoutConfig?: TimeoutConfig
  }

  interface AssigneeRule {
    type: 'fixed' | 'role' | 'department' | 'formField' | 'dynamic'
    value: string | string[]
    fallback?: string[]
  }

  // ---- 数据聚合 ----
  interface DataAggregation {
    id: string
    sourceForms: string[]
    aggregations: AggregationRule[]
    refreshInterval: number
    outputFormat: 'table' | 'chart' | 'api'
  }

  interface AggregationRule {
    sourceField: string
    operation: 'sum' | 'avg' | 'count' | 'max' | 'min' | 'group' | 'custom'
    targetField: string
    filter?: ConditionExpression
  }

  // ---- 集成网关 ----
  interface FormIntegration {
    type: 'finance' | 'salary' | 'crm' | 'external'
    event: 'onSubmit' | 'onApprove' | 'onReject' | 'onUpdate'
    mapping: FieldMapping[]
    endpoint: string
    retryPolicy?: RetryPolicy
  }

  interface FieldMapping {
    sourceField: string
    targetField: string
    transform?: 'none' | 'dateFormat' | 'numberFormat' | 'custom'
    transformConfig?: Record<string, unknown>
  }
}
```

## 12.3 AI 智能化增强

### 12.3.1 AI 表单自动生成

```typescript
/** AI 表单生成器 */
interface AIFormGenerator {
  /** 从自然语言描述生成表单 */
  generateFromDescription(description: string): Promise<SmartForm.FormDefinition>

  /** 从现有文档/图片识别生成表单 */
  generateFromDocument(file: File): Promise<SmartForm.FormDefinition>

  /** 从数据模型反向生成表单 */
  generateFromSchema(schema: Record<string, unknown>): Promise<SmartForm.FormDefinition>

  /** 智能推荐字段 */
  suggestFields(context: FormContext): Promise<FormFieldSuggestion[]>
}

interface FormFieldSuggestion {
  field: SmartForm.FormField
  confidence: number
  reason: string
  category: string
}
```

**使用场景示例**：

| 输入 | AI 输出 |
|:-----|:--------|
| "创建一个报销申请表单" | 自动生成：日期、金额、类别(下拉)、事由、附件上传、审批人 |
| "客户签约信息采集" | 自动生成：客户名、联系方式、合同金额、签约日期、附件 |
| "月度绩效考核表" | 自动生成：考核人、评分项(表格)、自评、主管评语 |

### 12.3.2 智能校验引擎

```typescript
/** 智能校验 — 超越规则引擎的语义级校验 */
interface SmartValidationEngine {
  /** 基于规则的校验 */
  validateRules(data: Record<string, unknown>, rules: ValidationRule[]): ValidationResult

  /** 基于AI的语义校验 */
  validateSemantics(data: Record<string, unknown>, context: FormContext): Promise<ValidationResult>

  /** 基于历史数据的异常检测 */
  detectAnomaly(data: Record<string, unknown>, historicalData: unknown[]): Promise<AnomalyResult>
}

interface ValidationResult {
  valid: boolean
  errors: ValidationError[]
  warnings: ValidationWarning[]
  suggestions: string[]
}

interface AnomalyResult {
  isAnomaly: boolean
  confidence: number
  explanation: string
  similarCases: unknown[]
}
```

## 12.4 审批流引擎

### 12.4.1 可视化审批流设计

```text
┌──────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────┐
│  提交节点  │───▶│  部门主管审批  │───▶│  财务审批     │───▶│  结束节点  │
│  (Start)  │    │  (Approval)   │    │  (Approval)  │    │  (End)   │
└──────────┘    └──────┬───────┘    └──────────────┘    └──────────┘
                       │
                       │ 金额 > 5000
                       ▼
                ┌──────────────┐
                │  总经理审批    │
                │  (Approval)   │
                └──────────────┘
```

### 12.4.2 条件分支规则

```typescript
interface ConditionBranch {
  condition: ConditionExpression
  trueNode: string   // 满足条件跳转的节点 ID
  falseNode: string  // 不满足条件跳转的节点 ID
}

interface ConditionExpression {
  field: string
  operator: 'eq' | 'neq' | 'gt' | 'gte' | 'lt' | 'lte' | 'contains' | 'in' | 'between'
  value: unknown
  logic?: 'and' | 'or'
  children?: ConditionExpression[]
}
```

### 12.4.3 审批动作类型

| 动作 | 说明 | 自动化能力 |
|:-----|:-----|:-----------|
| 同意 | 审批通过，流转下一节点 | 可配置自动通过条件 |
| 驳回 | 退回上一节点或指定节点 | 可配置驳回原因模板 |
| 转办 | 转交他人处理 | 支持转办理由记录 |
| 加签 | 增加临时审批人 | 支持前加签/后加签 |
| 会签 | 多人并行审批 | 配置通过比例（如 3/5） |
| 暂存 | 暂不处理，保留待办 | 超时自动提醒 |

```typescript
interface CrossFormAggregation {
```
